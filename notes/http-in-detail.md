What is HTTP?
(Hypertext Transfer Protocol) is the basic language for sending and receiving webpages and information on the internet.

**Key Concepts** 
HTTP Request   HTTP Response
Headers: Key-value pairs providing context (Content-Type, Content-Length, Authorization, or Caching rules).
Body: The actual payload (a JSON object, pdf or HTML document) sent with requests or responses.

Common HTTP Methods: GET(for getting information), POST(for submitting data and creating new records), PUT(for submitting data to update information),
DELETE(for deleting informations/records froma web server).

Common HTTP Status Codes: 200 - OK, 201 - created, 302 - found, 400 - bad request, 401 - not authorised, 403 - forbidden, 404 - page not found,
500 - internel service error, 503 - service unavaiable.

Cookie: Data sent to the server to help remember your information.
Cookies can be used for many purposes but are most commonly used for website authentication. 

HTTP is a stateless protocol. Each request is isolated, meaning the server retains no memory of previous requests.

HTTPS (HyperText Transfer Protocol Secure) protocol is a secure version of HTTP that encrypts web traffic with (tls/ssl) protocol to protect information by making it 
unreadable to third parties.
